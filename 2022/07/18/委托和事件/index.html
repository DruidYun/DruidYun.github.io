为什么要学习它

学习的原因，在实际的工程项目中在阅读他人的代码时，遇见了委托和事件。看到的时候一头雾水，不知道原理，使用方法，使用它能有什么方便的地方。每次看见它们俩，就憋的慌，因此来翻阅资料来了解。

委托和事件是什么

先谈谈委托的定义

委托在《C#入门经典》这本书第一次出现时出现在函数章中，此书对委托的描述是：

委托(delegate)是一种存储函数引用的类型。

委托在微软的C#编程指南中，是这样描述的：

委托是一种引用类型，表示对具有特定参数列表和返回类型的方法的引用。 在实例化委托时，你可以将其实例与任何具有兼容签名和返回类型的方法相关联。 你可以通过委托实例调用方法。。

在菜鸟教程中：

C# 中的委托（Delegate）类似于 C 或 C++ 中函数的指针。委托（Delegate） 是存有对某个方法的引用的一种引用类型变量。引用可在运行时被改变。

看到了前两种描述，并不是很清楚。直到看见了菜鸟的描述，类似于C中的函数指针。因为被C语言的指针折磨过了，大概是明白什么意思了。先用自己的了解的东西进行引申的确是个不错的方法。

调用委托

但是多播委托存在问题，委托会引用很多方法，如果是实例方法的话（非静态），也就是说它隶属与一个对象，如果我们引用这个方法的话，那么这个对象一定会存在内存中，即便没有其他引用变量引用这个对象，这个对象也不能进行释放，因为一旦释放，委托就不再能间接调用这个对象的方法了，所有委托可能导致内存泄漏。

怎么出现这个问题？

我们给照料植物的委托再给一个方法Fertilize(施肥)，但是不小心写成了=赋值符。这个时候问题就出现了。

这种缺点（将+=写成=）也是事件、观察者模式出现的原因。

Action委托和Func委托

这两种委托不需要提前声明，由C#库提供的两种内置泛型委托。

Action委托是返回值必须为空的委托，参数可以有也可以没有。

Func委托是必须设置返回值的委托，参数可以有也可以没有，并且返回值类型必须在<>中最后一位。

事件

首先委托是事件的基础，在C#中，事件由event关键词修饰。同样先讲讲在不同地方的描述吧。

在C#入门经典中，是这样说的：

事件类似于异常，因为它们都由对象引发(抛出)，并且都可以通过我们提供的代码来处理。

在C#编程指南中

类 或对象可以通过事件向其他类或对象通知发生的相关事情。 发送（或引发）事件的类称为“发布者”，接收（或处理）事件的类称为“订阅者”。

菜鸟教程的描述

事件（Event） 基本上说是一个用户操作，如按键、点击、鼠标移动等等，或者是一些提示信息，如系统生成的通知。应用程序需要在事件发生时响应事件。

事件由五个部分组成：

  • 事件的拥有者【对象】
  • 事件【event关键词修饰】
  • 事件的响应者【对象】
  • 事件的处理器【方法（受到约束的类型兼容的方法）】
  • 事件的订阅关系【+=】

事件必须由一个对象发出，由订阅这个事件的对象进行响应，并且进行处理。

比如手机(事件的拥有者)有闹钟响了这个事件，你(事件响应者)**订阅**了这个事件，然后你看了一眼手机，抱怨了一句还早，然后把闹铃关了，继续睡(事件处理器)。(这个例子不是十分的妥善，那就继续吧)

继续刚才的例子，你突然醒了想起今天还要上班，然后顾不得吃早饭，就匆匆出门了。可惜还是太晚了，你赶到了公司门口，看见了老板，不知道他为什么在门口等你。因为你(事件拥有者)迟到了(事件)，老板(事件响应者)专门在门口等你(订阅你迟到的事件)，最后夸了夸你，说你比上次迟到要早几分钟有进步，然后给记到了考勤上(事件处理器)。

当时间(事件拥有者)到了饭点(事件)，你和同事(两个事件响应者)注意到时间到了(订阅了时间)准备去吃饭，到了饭堂，同事要了一份正常的套餐(事件处理器)，而你因为早上没吃饭，想着加餐，在正常的套餐的基础上多要了一些米多要了个鸡腿(事件处理器)。

事件和委托是什么关系

在反编译中，可以发现简略声明格式的情况时，自动生成了一个委托类型的字段(在后台，访问不到也看不到），用来存储事件处理器，就像完整声明格式里的一样。这种结构就像属性一样，是一种包装器，是面向对象的封装的体现，属性保护着字段不被非法的操作所污染，不被滥用，属性是字段的包装器，而事件保护着委托类型字段，不被外界滥用，只能获取它的+=，-=操作符，是委托类型字段的包装器。属性不是字段，而事件也不是委托类型字段。

事件让程序更加安全，更好的维护。

我能用委托和事件来干什么

委托是事件的基础，也是Lambda表达式的基础。在使用这些的时候都是在使用委托，另外在游戏开发中，有很多功能可以使用事件很方便的写出来，比如角色的死亡，在这件事件触发的时候会有很多地方需要依次做各种事情。还有到达某个区域触发一些事件，比如打开门，刷怪，进入对话等等的功能都可以用事件的订阅，回调来执行。
