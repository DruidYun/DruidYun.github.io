AI学习(新坑)

发表于 2026-03-05 分类于 打卡
本文字数： 833 阅读时长 ≈ 1 分钟

前言，新的一年开始了，曾经感觉的泡沫感觉突然已经成为一股大浪🌊🌊🌊，可能是近段时间对科技方面关注点太少了，现在它们来了，要么我迎上它，成为勇敢的弄潮儿，要被站在岸边被冲刷到粉身碎骨。也罢反正我也不想一直干一样的事，也总会想如果有一天我失去当前身份，不再写程序会成为什么样子，我还能做什么，与其一直思考不如借这次的思想这次的东风，来试一试。

最初的目标

关于我到底要做些什么，学什么内容，和到达什么样的阶段性目标，在3月4日5日这两天，我订阅了Plus会员，和chatgpt5进行了深入交流。最后以AI数字分身，AI工具链以及AIGC的创作为目标，既然这一节叫做最初的目标，所以这个目标极大概率是会发生改变的，不过总要有个开始。
为了达成这两目标，还得有个先后顺序，以及内容需要了解。目前和AI的沟通主要在Python，RAG及模型微调上，也就是更倾向于AI数字分身。对于其他两点，AIGC之前有玩过文生图，图生图，一些小小的尝试，但是也是主要在应用上进行过非常小的尝试，甚至没有弄清一些复杂的功能。那个AI工具链了解的更少了，对于它的了解都是来自千问的送奶茶和之前认识的做UE的人一个让我焦虑的朋友圈。
我先以AI数字分身这个方向，开始学习，可能会对AIGC后续的模型调试会有帮助，虽然我也知道LLM模型和那些图像模型一定有很大的差别。

AI数字分身

学习内容

主要内容以Python、、embedding 、RAG、AI应用开发为主，AI不推荐学习模型微调说微调成本高，微调一个 7B 模型通常需要几十万条数据，数小时到数天 。目前的大部分产品主要是LLM、RAG、Prompt为主。

Python

要开始学Python，上次学Python还是大学做无人机方面的时候要做一个图像处理的时候，有一定学习，也没学多少，gpt告诉我要学习AI这方面不需要要学习太多的基础，先把关键的学了，变量、列表、字典、函数、循环这些关键的基础再其他语言也有，也不多加赘述了。现在直接从库的使用开始学习。
