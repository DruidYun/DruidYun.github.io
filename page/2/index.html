CPoint的y值为0的问题

发表于 2021-09-18 更新于 2023-01-25 分类于 笔记
本文字数： 234 阅读时长 ≈ 1 分钟

今天突然遇到一个奇怪的问题，就无论CPoint的点在什么位置，在mfc的窗体一直都在最上方画出一条横线。也就是y等于0.

然后在Debug中发现只要使用p = （200,200);这样赋值的话，y值就会是0.

最终发现**p = (200,200);**中(200,200)本身是一段逗号表达式，表达式的值为200。它的构造方法并没有对赋值进行重载，

解决方法是:CPoint p(200,200);这样使用构造方法直接赋值。

另外一种就是单独用p.x = 200和p.y = 200进行赋值。

双缓冲绘图算法

发表于 2021-09-17 更新于 2023-01-25 分类于 笔记
本文字数： 966 阅读时长 ≈ 1 分钟

定时器

双缓冲是指一个显示缓冲区（显示设备上下文）和一个内存缓冲区（内存设备上下文）。双缓冲机制是一个基本的动画技术，常用于解决单缓冲擦除图像时带来的屏幕闪烁问题。

单缓冲是直接将图形绘制到了显示缓冲区。

双缓冲是先将图像绘制到内存缓冲区，再从内存缓冲区一次性将图像拷贝到显示缓冲区。

定时器函数

原型：UINT SetTimer(UINT nIDEvent, UINT nElapse, void (CALLBACKEXPORT* IpfnTimer)(HWND, UINT, UINT, DWORD))

参数：

  • nIDEvent是非零的定时器标识符
  • nElapse是以毫秒表示的时间间隔
  • pfnTimer是处理WM_TIMER消息的TimerProc()回调函数的地址

原型：BOOL KillTimer(int nIDEvent);

参数：nIDEvent是由SetTimer函数设置的定时器标识符
